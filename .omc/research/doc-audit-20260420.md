## Notable gaps (prioritized)

### Gap 1 — Pain-point 5 × 4 docs (mcp 1.27.0 compat: FastMCP description + return type fix)
**Priority: HIGHEST.** This is a silent breaking change that causes the MCP server to fail at runtime with no obvious error. It is documented only in `.memory/MEMORY.md:28` (one sentence). None of the four setup/quickstart/guide/agents docs contain the specific fix. Any new developer following ENVIRONMENT_SETUP.md or QUICKSTART.md will clone `isaac-sim-mcp`, attempt to run it as-is, and hit a runtime failure with no guidance.
- Affects: ENVIRONMENT_SETUP.md, QUICKSTART.md, robot-dev-omc-setup-guide.md, AGENTS.md
- Remediation: Add a "server.py patch" subsection to ENVIRONMENT_SETUP.md §2 (Isaac Sim MCP 연동) with the exact two changes: remove `description=` kwarg from `FastMCP()` and fix the return type annotation. Cross-reference in QUICKSTART.md troubleshooting section.

### Gap 2 — Pain-point 3 × 3 docs (Docker MCP deferred — decision path not recorded in ENVIRONMENT_SETUP, QUICKSTART, AGENTS)
**Priority: HIGH.** The deferral decision for Docker MCP is only implicit. `robot-dev-omc-setup-guide.md` section 3/10 and `MEMORY.md` MCP config section together imply it, but `ENVIRONMENT_SETUP.md`, `QUICKSTART.md`, and `AGENTS.md` contain no explicit "Docker MCP: deferred — see robot-dev-omc-setup-guide.md §3" note. Future developers will search these primary docs and find nothing, potentially re-implementing it incorrectly.
- Affects: ENVIRONMENT_SETUP.md, QUICKSTART.md, AGENTS.md
- Remediation: Add a one-line "Docker MCP: 미구현 (Non-Goal Phase 1) — robot-dev-omc-setup-guide.md §3 참조" note in ENVIRONMENT_SETUP.md §2 and AGENTS.md MCP section.

### Gap 3 — Pain-point 2 × 2 docs (WebRTC AppImage install: robot-dev-omc-setup-guide.md, AGENTS.md)
**Priority: MEDIUM.** `robot-dev-omc-setup-guide.md` and `AGENTS.md` both omit any mention of the WebRTC Streaming Client or the 127.0.0.1:49100 connection procedure. Since `robot-dev-omc-setup-guide.md` is the primary OMC/env-setup document a new developer on the `~/robot` migration path will read, the gap means they will not know how to visually verify Isaac Sim is running.
- Affects: robot-dev-omc-setup-guide.md, AGENTS.md
- Remediation: Add a "Isaac Sim 시각 확인" bullet to AGENTS.md §환경 시작 방법 and a note in robot-dev-omc-setup-guide.md §12 (개발환경 실전 교훈).
