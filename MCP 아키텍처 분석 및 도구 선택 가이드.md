# MCP 성능 최적화 및 스킬 추천 (업데이트 버전)

현재 사용 중인 `omni-mcp/isaac-sim-mcp`와 `robotmcp/ros-mcp-server`는 매우 훌륭한 범용 도구입니다. 하지만 LIMO PRO 프로젝트의 "실시간성"과 "토큰 효율성"을 극대화하기 위해 다음의 고도화된 도구들을 추천합니다.

---

## 1. MCP 서버 교체 및 보완 (성능 및 토큰 최적화)

### **Isaac Sim 제어 최적화**
*   **추천: `nullbyte91/nvidia-isaac-mcp`**
    *   **특징**: `execute_script`를 통한 통짜 코드 주입 대신, `get_prim_info`, `create_prim` 등 세분화된 CRUD 도구를 제공합니다.
    *   **이점**: 모델이 필요한 부분만 로드하게 함으로써 컨텍스트 오버헤드를 최대 90% 이상 절감할 수 있으며, 4.5.0 API 환경에서도 안정적인 객체 조작이 가능합니다.
*   **대안: `whats2000/isaacsim-mcp-server`**
    *   **특징**: `omni-mcp`의 개선된 포크 버전입니다.
    *   **이점**: 다양한 로봇 자산(Jetbot, Franka 등)에 대한 사전 정의된 접근을 지원하며, 시뮬레이션 상태를 다시 에이전트에게 전달하는 피드백 루프 형성에 유리합니다.

### **ROS 2 통신 고도화**
*   **추천: `wise-vision/ros2_mcp`**
    *   **특징**: ROS 2의 토픽, 서비스, 액션을 AI 에이전트가 직접 발견하고 상호작용할 수 있는 고수준 브리지를 제공합니다.
    *   **이점**: 단순한 메시지 전송을 넘어, 복잡한 로봇 동작 시퀀스를 AI가 직접 오케스트레이션하고 실시간으로 상태를 모니터링하는 데 최적화되어 있습니다.

---

## 2. 필수 도입 스킬 및 도구 (이미 도입된 항목 포함)

### **[도입 완료] 절차적 지식의 분리 (`arpitg1304/robotics-agent-skills`)**
*   **상태**: **현재 프로젝트에 적용됨.**
*   **역할**: ROS 2 개발 패턴, DDS 설정 등 로보틱스 전용 지침을 SKILL.md 형태로 분리하여 에이전트의 결정 능력을 높이고 컨텍스트 오염을 방지합니다.

### **[추천] 도구 동적 로딩 (`philschmid/mcp-cli`)**
*   **역할**: 에이전트에게 너무 많은 MCP 도구를 직접 연결하는 대신, 필요할 때만 CLI를 통해 도구를 검색하고 호출하는 방식(Lazy Loading)을 지원합니다.
*   **이점**: 세션 시작 시 수만 개의 토큰을 소모하는 '도구 스키마 로드' 비용을 획기적으로 줄여줍니다. 또한 개발 중 MCP 서버의 응답 정합성을 에이전트 없이 독립적으로 테스트할 수 있습니다.

---

## 3. 요약 및 권장 로드맵

1.  **1단계 (토큰 최적화)**: `nullbyte91` 서버를 도입하여 Isaac Sim 객체 조작 시 발생하는 토큰 낭비를 줄입니다.
2.  **2단계 (성능 최적화)**: `wise-vision` 서버를 통해 ROS 2 기반의 실시간 제어 시퀀스를 고도화합니다.
3.  **3단계 (테스트 자동화)**: `mcp-cli`를 활용하여 데이터 생성 파이프라인의 각 단계를 독립적으로 검증하는 루틴을 구축합니다.

> **참고**: `mcp-debug-tools`는 현재의 복잡한 로보틱스 환경에서는 오버헤드가 클 수 있어 도입 대상에서 제외되었습니다.
